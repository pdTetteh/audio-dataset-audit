## AudioDatasetAudit v0.2.0

This release adds the first major expansion of AudioDatasetAudit beyond the initial scaffold.

### Added
- leakage checks for:
  - `speaker_id`
  - `device_id`
  - `date`
  - `location`
- HTML report generation
- example leaky manifest
- example Markdown and HTML reports
- tests for leakage detection
- updated README examples that match the real CLI

### Why this matters
Speech and audio datasets often look healthy on the surface while still containing silent problems that can distort evaluation results. This release makes it easier to catch:
- train/validation/test leakage
- metadata overlap across splits
- dataset hygiene issues before model training

### Current capabilities
AudioDatasetAudit can now:
- validate manifest structure
- report missing metadata
- report class imbalance
- detect duplicate IDs and file paths
- validate split integrity
- detect leakage across splits
- generate JSON, Markdown, and HTML reports

### Example usage
```bash
python3 -m audiodatasetaudit.cli examples/leaky_manifest.csv --format html --output examples/leaky_report.html
```

### Next
The next milestone is file-level audio probing and quality checks, including:
- missing or unreadable files
- sample-rate mismatch
- channel mismatch
- empty files
- duration anomalies
